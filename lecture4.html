Understanding File Path System in HTML

When we add images, links, or files in HTML, we need to tell the browser where the file is located. This is done using file paths.

Types of File Paths

  1. Absolute Path
    • Full location of a file (includes domain name).
    • Used when file is on another website/server.
    • For Example: < img src="https://example.com/images/pic.jpg" alt="Example">
  2. Relative Path
    • Location of a file relative to your HTML file.
    • Common in projects because all files are inside folders.
    • Here, browser looks for pic.jpg inside images folder (same project).
    • For Example: < img src="images/pic.jpg" alt="Example">
  3. Root-relative Path
    • Starts from the root directory of the website.
    • Always begins with /.
    • For Example: < img src="/assets/images/pic.jpg" alt="Example">

Header, Main, Footer in HTML

  1. header
    • Top section of a page or a section.
    • Mostly contains logo, navigation menu, heading.
    • Can be used multiple times (like page header or article header).
  2. main
    • The main content of the webpage.
    • There should be only ONE < main> per page.
    • Excludes header, footer, sidebar, etc.
  3. footer
    • Bottom section of a page or a section.
    • Contains copyright, links, contact info, social media.
In Short:
  • Header = top part (logo, nav, title).
  • Main = middle (main unique content).
  • Footer = bottom (copyright, links).

What is article Tag in HTML?

  • < article> is a semantic HTML5 tag.
  • Used for independent, self-contained content.
  • Can be shared or reused separately (like blog posts, news articles, comments, forum posts).
