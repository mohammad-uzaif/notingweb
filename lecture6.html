What is the < media> tag in HTML?

Actually, there is no HTML element called < media>.

But the term "media" in HTML usually refers to media-related elements (like audio, video, images) and media queries in CSS. Let me clarify both so you don’t get confused:

Media Elements in HTML

  • < img> → For images
  • < audio> → For sound/music
  • < video> → For videos
  • < source> → Defines multiple media sources for < audio> or < video>
  • < track> → Subtitles/captions in video or audio

HTML Media Elements - Important Attributes

This page explains the main attributes used in <audio> and <video> tags.

1. controls

Shows built-in browser controls (play, pause, volume, etc.).

<audio controls>
  <source src="song.mp3" type="audio/mpeg">
</audio>

2. autoplay

Starts playing automatically when the page loads ( often requires muted).

<video autoplay muted>
  <source src="movie.mp4" type="video/mp4">
</video>

3. muted

Starts media with the sound turned off.

<audio autoplay muted loop>
  <source src="background.mp3" type="audio/mpeg">
</audio>

4. loop

Plays the media repeatedly after it ends.

<video controls loop>
  <source src="clip.mp4" type="video/mp4">
</video>

5. preload

Suggests how much of the file the browser should load before playback.

  • none → don’t preload
  • metadata → load only info (length, size)
  • auto → load entire file
<audio controls preload="metadata">
  <source src="music.mp3" type="audio/mpeg">
</audio>

6. poster (video only)

Shows a preview image before the video starts.

<video controls poster="preview.jpg">
  <source src="trailer.mp4" type="video/mp4">
</video>

7. src

Directly adds the media file (alternative to using <source>).

<audio controls src="song.mp3"></audio>

8. crossorigin

Handles cross-origin requests for media files (used for external files).

<video controls crossorigin="anonymous">
  <source src="movie.mp4" type="video/mp4">
</video>

9. width / height (video only)

Sets the size of the video player.

<video controls width="400" height="300">
  <source src="movie.mp4" type="video/mp4">
</video>

Quick Summary

  • Common: controls, autoplay, muted, loop, preload, src, crossorigin
  • Video only: poster, width, height

References: MDN - <video> element | MDN - <audio> element

How to Deploy a Website on Netlify

This page explains the step-by-step process to deploy your HTML project on Netlify.

Step 1: Prepare Your Project

  1. Put all your files in a single project folder.
  2. Ensure the main file is named index.html (Netlify looks for it by default).
my-website/
   index.html
   about.html
   style.css
   script.js
   images/

Step 2: Create a Netlify Account

  1. Go to https://www.netlify.com
  2. Sign up with GitHub, GitLab, Bitbucket, or your email.

Step 3: Deploy by Drag and Drop (Quick Method)

  1. Login to your Netlify Dashboard.
  2. Click Add new site → Deploy manually.
  3. Drag and drop your project folder (with index.html inside).
  4. Netlify will give you a live link like:
    https://your-project-name.netlify.app

Step 4: Deploy Using GitHub (Recommended Method)

  1. Push your project to a GitHub repository (example: my-website).
  2. In Netlify Dashboard, click Add new site → Import from Git.
  3. Select GitHub and authorize access.
  4. Choose your repository and branch (usually main).
  5. Netlify will build and deploy automatically.

Step 5: Manage Your Site

  • You can rename your site URL in Site Settings → Change site name.
  • If using GitHub, every push automatically redeploys your site.

Common Issues

  • If you see "Page not found":
    • Make sure index.html is in the root folder.
    • Check your links. Use ./about.html instead of absolute paths.
